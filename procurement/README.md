## Roles and Responsibilities

| Role | Primary Responsibilities |
|------|--------------------------|
| **Site Supervisor** | Raises purchase requests specifying type, quantity, and purpose. Verifies materials upon delivery at site. |
| **Procurement/Office** | Coordinates approvals, sources suppliers, manages supplier evaluation process. |
| **Managing Director (MD)** | Approves/adjusts/rejects purchase requests. Final approval on supplier selection. |
| **Site Technician/Artisan (Fundi)** | Assists in physical inspection and verification of delivered materials. |
| **Store Keeper** | Maintains material control and updates stock records daily per site. |

**Key Fields:**

- Project/Site name
- Material description, specifications, quantity
- Purpose/justification
- Required delivery date

### 2. Quotation Comparison Sheet
**File:** `Quotation_Comparison_Sheet.docx`

**Purpose:** Compare and evaluate quotations from multiple suppliers (minimum 3 required).

**Key Fields:**

| Field | Description |
|-------|-------------|
| Project/Site Name | The project requiring materials |
| Purchase Request Ref No | Reference to original request |
| Date | Date of comparison |

**Supplier Comparison Table:**

| Column | Description |
|--------|-------------|
| Supplier Name | Name of quoting supplier |
| Quoted Price | Total quoted amount |
| Delivery Time | Promised delivery timeline |
| Material Quality | Assessment of material quality |

| Remarks | Additional notes |
| Selected (Yes/No) | Final selection indicator |

**Recommendation Section:**
- Recommended Supplier name
- Reason for Selection (Price / Quality / Reliability)

**Required Signatures:**
1. Procurement Officer (Prepared by)
2. Managing Director (Approved by)

## Detailed Process Steps

### Step 1: Purchase Request
**Responsible:** Site Supervisor

**Actions:**
1. Identify material requirements for the project
2. Complete Purchase Request Form with:
   - Material type and specifications
   - Required quantity
   - Purpose/justification
   - Preferred delivery date
3. Submit to Procurement/Office

**Output:** Completed Purchase Request Form

---

### Step 2: Management Approval
**Responsible:** Managing Director

**Actions:**
1. Review purchase request for:
   - Budget alignment
   - Project necessity
   - Quantity reasonableness
2. Decision options:
   - **Approve** - Proceed as requested
   - **Adjust** - Modify quantities or specifications
   - **Reject** - Deny with reason

**Output:** Approved/Adjusted/Rejected Purchase Request

---

### Step 3: Feedback
**Responsible:** Procurement/Office

**Actions:**
1. Communicate MD's decision to Site Supervisor
2. If adjusted, clarify changes
3. If rejected, provide reason

**Output:** Decision communicated to requestor

---

### Step 4: Supplier Sourcing
**Responsible:** Procurement/Office

**Actions:**
1. Identify minimum 3 approved suppliers
2. Send Request for Quotation (RFQ) to each supplier
3. Collect quotations with:
   - Itemized pricing
   - Delivery timeline
   - Payment terms
   - Validity period

**Output:** Minimum 3 supplier quotations

---

### Step 5: Evaluation & Recommendation
**Responsible:** Procurement/Office

**Actions:**
1. Complete Quotation Comparison Sheet
2. Evaluate suppliers on:
   - **Price** - Total cost including VAT
   - **Quality** - Material specifications and standards
   - **Reliability** - Past performance, delivery track record
   - **Payment Terms** - Credit period, payment methods
3. Document recommendation with justification

**Output:** Completed Quotation Comparison Sheet with recommendation

---

### Step 6: Supplier Approval and Purchase
**Responsible:** Managing Director + Procurement

**Actions:**
1. MD reviews comparison sheet and recommendation
2. MD approves final supplier selection
3. Procurement issues Purchase Order to selected supplier
4. Payment processed per agreed terms

**Output:** Approved Purchase Order

---

### Step 7: Delivery to Site
**Responsible:** Supplier + Site Supervisor

**Actions:**
1. Supplier delivers materials to designated project site
2. Supplier representative presents Delivery Note
3. Site Supervisor receives and signs Delivery Note

**Output:** Signed Delivery Note

---

### Step 8: Inspection and Verification
**Responsible:** Site Supervisor + Fundi/Technician

**Actions:**
1. Physical inspection of delivered materials
2. Verify:
   - Quantity matches order and delivery note
   - Quality meets specifications
   - No damage or defects
3. Complete Inspection Record (Part A)
4. Determine acceptance status:
   - Accepted
   - Partially Accepted (note rejected items)
   - Rejected (document reasons)
5. Complete Received Material Document (GRN)

**Output:** Completed Inspection Record and GRN

---

### Step 9: Stock Update
**Responsible:** Store Keeper/Site Supervisor

**Actions:**
1. Update Site Stock Record (Part B of Inspection form)
2. Record:
   - Opening stock
   - Quantity received
   - Calculate closing stock
3. Update inventory system
4. File all documentation

**Output:** Updated Stock Records

---

## Compliance Requirements

### Mandatory Requirements

1. **Minimum 3 Quotations:** All purchases must have documented quotations from at least 3 suppliers
2. **MD Approval:** All purchase requests require Managing Director approval
3. **Document Trail:** Complete documentation chain must be maintained:
   - Purchase Request → Quotations → Comparison Sheet → PO → Delivery Note → Inspection Record → GRN → Stock Update
4. **Dual Verification:** Material inspection requires both Site Supervisor and Fundi signatures
5. **Same-Day Stock Update:** Store Keeper must update stock records daily

### Deviation Handling

Any deviation from this procedure must be:
1. Documented in writing
2. Approved by management
3. Filed with related procurement documents

### Approval Workflow

The ERP uses **RingleSoft Laravel Process Approval** for multi-level approvals:

```
Purchase Request Created
        ↓
    [PENDING]
        ↓
   MD Reviews
        ↓
  ┌─────┴─────┐
  ↓           ↓
[APPROVED] [REJECTED]
```

   - Add inspection checklist fields
   - Condition tracking (Good/Damaged/Short/Excess)
   - Photo upload for damaged items

3. **Stock Integration**
   - Auto-update `ProjectMaterialInventory` on GRN approval
   - Real-time stock level alerts
