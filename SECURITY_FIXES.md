### Fix #2 — IDOR on Engineering Design controllers (HIGH)
**Files:**
- `app/Http/Controllers/Api/V1/StructuralDesignApiController.php`
- `app/Http/Controllers/Api/V1/ServiceDesignApiController.php`

Both controllers already check `hasAnyRole(['Managing Director','CEO','Chief Executive Officer','System Administrator','Admin'])` for the index "see all" filter. The same set should gate per-id endpoints. Affected methods: `show`, `update`, `destroy`, `updateStage`, `submitStage`.

Add this private helper to each controller (use `$request->user()` where the method receives a Request; pass it in for `show`/`destroy` by changing the signature to `public function show(Request $request, int $id)`):

```php
private const MANAGER_ROLES = [
    'Managing Director', 'CEO', 'Chief Executive Officer',
    'System Administrator', 'Admin',
];

private function ensureCanAccess(Request $request, $design): void
{
    $user = $request->user();
    $isManager = $user && $user->hasAnyRole(self::MANAGER_ROLES);
    abort_unless(
        $isManager || $design->assigned_engineer_id === $user?->id,
        403,
        'You are not authorized to access this design.',
    );
}
```

Then, immediately after each `findOrFail($id)`:

```php
$this->ensureCanAccess($request, $design);
```

In `update()` also gate the engineer-reassignment field:

```php
$validated = $request->validate([
    'assigned_engineer_id' => 'nullable|exists:users,id',
    'notes' => 'nullable|string|max:1000',
]);

// Engineers cannot reassign themselves — managers only.
$isManager = $request->user()->hasAnyRole(self::MANAGER_ROLES);
if (!$isManager) {
    unset($validated['assigned_engineer_id']);
}
$design->update($validated);
```

Optionally, restrict `destroy()` to managers altogether (engineers usually can't delete their own design rows):

```php
abort_unless($isManager, 403, 'Only managers can delete designs.');
```

A cleaner long-term fix is a Policy registered in `AuthServiceProvider` (`ProjectStructuralDesignPolicy`, `ProjectServiceDesignPolicy`) and `$this->authorize('view', $design)` etc. — but the inline guard above closes the IDOR window today.
